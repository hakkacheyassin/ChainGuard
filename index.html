ChainGuard

Supply Chain Security Scanner · Documentation

ChainGuard is a full-stack platform that analyzes the software supply chain of any project. Paste a GitHub or GitLab repository URL and it clones the code, builds a complete SBOM (Software Bill of Materials), matches every dependency against known CVE vulnerability databases, and summarizes license compliance — all automatically, on any technology stack (JavaScript, Python, Go, Java, Rust, PHP and more).

Powered by Syft & Grype

01. How the app works

STEP 1

Paste a repo URL

GitHub or GitLab, public or private (with an access token).

STEP 2

Confirm the stack

Languages, stars, size and branch are detected automatically and shown for confirmation.

STEP 3

Live scan phases

Clone → SBOM generation (Syft) → vulnerability matching (Grype), with real-time progress.

STEP 4

Download the report

A completion popup offers the SBOM (CycloneDX, SPDX) and the report (JSON, CSV).

03. The pages in detail

01

Security Dashboard

The home of every scan: total packages, vulnerability count, an animated security score gauge (0–100), findings by severity, the top vulnerabilities table, license summary and a live graph of the most vulnerable dependencies of the last scanned project.

02

Dependency Tree

An interactive graph of the scanned project. Vulnerable packages appear first with severity colors and animated edges. Clicking any package opens a panel with its version, license, CVE list, fix versions and dependency path. A project selector switches between all scanned repositories.

03

Vulnerabilities

The complete list of findings from the latest scan, filterable by severity (Critical / High / Medium / Low) and searchable by CVE ID or package name. Each row shows the installed version and the version that fixes the issue.

04

Licenses

License compliance at a glance: a distribution chart plus a table grouping every package by its license — essential for legal review before shipping. Includes a project selector too.

05

History

Every scan the user has run, with its status (completed / failed with the exact error), findings summary, security score and one-click downloads: SBOM in CycloneDX or SPDX, report in CSV or JSON.

06

Analyze SBOM

Already have an SBOM from another tool or a vendor? Drag and drop the CycloneDX or SPDX file: the format is detected automatically and a report is generated instantly — packages, license breakdown and exports.

07

SBOM Explorer

A searchable table of every package detected in the scan: name, exact version, license and vulnerability status per package.

08

Settings

GitHub and GitLab access tokens for private repositories, each with a Test button that verifies the token against the provider and shows the connected account. Plus scan preferences.
